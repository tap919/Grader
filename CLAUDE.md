## Daily Protocol (Mandatory)
After each session, you MUST update the acquisition tracker:
1. **`.acquisition-tracker/DAILY-LOG.md`** — Record work done, key decisions, blockers, next actions. Create a new dated entry for each session.
2. **`.acquisition-tracker/PROGRESS.md`** — Update deploy readiness %, overall progress, sprint backlog items completed.
3. **`.acquisition-tracker/INSIGHTS.md`** — Log any market signals, competitive intelligence, or strategic opportunities detected during the session.
4. **`.acquisition-tracker/METRICS.md`** — Update scores if evidence supports a change.

## Session Startup
1. Read `.acquisition-tracker/DAILY-LOG.md` latest entry → understand last state
2. Read `.acquisition-tracker/AGENDA.md` → align work with mission
3. Read `.acquisition-tracker/PROGRESS.md` → know current sprint backlogs
4. Begin work
